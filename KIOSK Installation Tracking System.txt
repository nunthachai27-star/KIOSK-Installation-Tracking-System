# Top Level Specification (TLS)

## ระบบบันทึกและติดตามงานติดตั้ง KIOSK

KIOSK Installation Tracking System

---

## 1. Project Overview

### 1.1 ชื่อระบบ

ระบบบันทึกและติดตามงานติดตั้ง KIOSK
KIOSK Installation Tracking System

### 1.2 วัตถุประสงค์ของระบบ

พัฒนาระบบสำหรับบริหารจัดการงานติดตั้ง KIOSK แบบครบวงจร โดยนำข้อมูลจาก Excel เดิมมาเป็นข้อมูลตั้งต้น และเปลี่ยนกระบวนการทำงานจากการบันทึกหลายชีตใน Excel ให้เป็นระบบฐานข้อมูลกลาง

ระบบต้องรองรับการทำงานของ 3 กลุ่มหลัก ได้แก่

1. งานธุรการ
   บันทึกข้อมูลตั้งต้น สัญญา ใบสั่งซื้อ เอกสารส่งมอบ และการเปิดใบแจ้งหนี้

2. ทีมช่าง
   บันทึกรายละเอียดสินค้า อุปกรณ์ Serial Number และผลการตรวจสอบ QC

3. ทีมติดตั้ง
   บันทึกข้อมูลการจัดส่ง การ Remote ติดตั้ง การติดตั้งหน้างาน การอบรม Checklist และปัญหาที่พบ

ระบบต้องรองรับการใช้งานผ่าน

* Desktop / Notebook
* Tablet
* Mobile Phone

ในรูปแบบ Responsive Web Application ตั้งแต่ระยะแรก

## 2. Business Objective

### 2.1 เป้าหมายทางธุรกิจ

ระบบนี้มีเป้าหมายเพื่อยกระดับการติดตามงาน KIOSK จาก Excel Tracking ให้เป็น Operation Management System ที่สามารถติดตามงานได้ตั้งแต่เริ่มต้นจนถึงปิดงาน

### 2.2 ผลลัพธ์ที่ต้องการ

* รวมข้อมูล KIOSK ทั้งหมดไว้ในระบบเดียว
* ลดการใช้ Excel หลายชีต
* ลดข้อมูลซ้ำและข้อมูลตกหล่น
* ติดตามสถานะงานได้แบบ Real-time
* เห็นแผนงานประจำสัปดาห์ชัดเจน
* รู้ว่าสัปดาห์นี้ต้องส่งของ ติดตั้ง หรืออบรมที่ใด
* ติดตาม Serial Number ได้ครบถ้วน
* ติดตามงานค้าง Checklist / ส่งมอบ / เปิดใบแจ้งหนี้ ได้ง่าย
* รองรับการใช้งานภาคสนามผ่านมือถือและ Tablet
* ผู้บริหารสามารถดู Dashboard เพื่อบริหารงานได้ทันที

## 3. System Scope

### 3.1 In Scope

ระบบต้องรองรับความสามารถหลักดังนี้

1. Login และกำหนดสิทธิ์ผู้ใช้งาน
2. Import ข้อมูลจาก Excel เดิม
3. จัดการข้อมูลงานติดตั้ง
4. จัดการข้อมูลโรงพยาบาล
5. จัดการข้อมูลผู้ติดต่อโรงพยาบาล
6. จัดการข้อมูลสัญญา / ใบสั่งซื้อ
7. จัดการข้อมูลสินค้าและอุปกรณ์
8. บันทึก Serial Number
9. ตรวจสอบ Serial Number ซ้ำ
10. บันทึกผล QC สินค้า
11. บันทึกการจัดส่งสินค้า
12. บันทึกการ Remote ติดตั้ง
13. บันทึกการติดตั้งหน้างาน
14. บันทึกการอบรม
15. บันทึกปัญหาและวิธีแก้ไข
16. บันทึก Checklist
17. บันทึกเอกสารส่งมอบงาน
18. บันทึกสถานะการเปิดใบแจ้งหนี้
19. Dashboard แผนงานประจำสัปดาห์
20. รายงานสำหรับผู้บริหาร
21. แจ้งเตือนงานค้าง / งานใกล้ครบกำหนด
22. รองรับ Mobile และ Tablet
23. Export รายงานเป็น Excel / PDF

### 3.2 Out of Scope ระยะแรก

รายการที่ยังไม่จำเป็นต้องพัฒนาใน Phase แรก

* ระบบบัญชีเต็มรูปแบบ
* ระบบ Stock / Inventory แบบสมบูรณ์
* ระบบ GPS Tracking ขนส่ง
* Native Mobile Application
* ระบบอนุมัติเอกสารหลายระดับ
* เชื่อมต่อระบบบัญชีภายนอก
* เชื่อมต่อ HOSxP โดยตรง

## 5. High Level Workflow

### 5.1 Workflow หลักของระบบ

1. ธุรการ Import ข้อมูลจาก Excel เดิม
2. ระบบอ่านข้อมูลจากหลายชีตและแปลงเป็นรูปแบบกลาง
3. ธุรการตรวจสอบข้อมูลโรงพยาบาล จังหวัด จำนวน ยอดขาย และเอกสาร
4. ธุรการบันทึกข้อมูลสัญญา ใบสั่งซื้อ และกำหนดส่งมอบ
5. ระบบส่งงานต่อให้ทีมช่าง
6. ทีมช่างบันทึกรายละเอียดสินค้าและ Serial Number
7. ทีมช่างบันทึกผล QC
8. เมื่อ QC ผ่าน ระบบเปลี่ยนสถานะเป็นพร้อมจัดส่ง
9. ทีมติดตั้งบันทึกข้อมูลจัดส่ง
10. ทีมติดตั้งกำหนดวัน Remote / Onsite
11. ทีมติดตั้งบันทึกผลติดตั้ง
12. ทีมติดตั้งบันทึกผลอบรม
13. ทีมติดตั้งแนบ Checklist และรูปหน้างาน
14. ธุรการตรวจสอบเอกสาร Checklist
15. ธุรการบันทึกส่งมอบงาน
16. ธุรการบันทึกเปิดใบแจ้งหนี้
17. ระบบเปลี่ยนสถานะเป็นปิดงาน

## 13. Dashboard Logic

### 13.1 Logic การดึงงานประจำสัปดาห์

Dashboard ต้องดึงข้อมูลจาก `job_activities` เป็นหลัก โดยใช้ `activity_date` อยู่ในช่วงสัปดาห์ที่เลือก

### 13.2 ตัวอย่าง Logic

* ถ้า activity_type = ส่งของ และ activity_date อยู่ในสัปดาห์นี้ ให้แสดงในกลุ่มงานส่งของ
* ถ้า activity_type = Remote ติดตั้ง และ activity_date อยู่ในสัปดาห์นี้ ให้แสดงในกลุ่ม Remote
* ถ้า activity_type = ติดตั้งหน้างาน และ activity_date อยู่ในสัปดาห์นี้ ให้แสดงในกลุ่ม Onsite
* ถ้า activity_type = อบรม และ activity_date อยู่ในสัปดาห์นี้ ให้แสดงในกลุ่มอบรม
* ถ้างานเลยกำหนดส่งมอบและยังไม่ปิดงาน ให้แสดงในกลุ่ม Overdue
* ถ้าติดตั้งแล้วแต่ยังไม่มี Checklist ให้แสดงในกลุ่มรอ Checklist
* ถ้าส่งมอบแล้วแต่ยังไม่เปิดใบแจ้งหนี้ ให้แสดงในกลุ่มรอเปิดใบแจ้งหนี้

## 20. Summary

ระบบบันทึกและติดตามงานติดตั้ง KIOSK เป็นระบบกลางสำหรับบริหารจัดการงาน KIOSK ตั้งแต่การนำเข้าข้อมูลจาก Excel เดิม ไปจนถึงการเตรียมสินค้า จัดส่ง ติดตั้ง อบรม ส่งมอบงาน และเปิดใบแจ้งหนี้

หัวใจสำคัญของระบบคือการใช้ “งานติดตั้ง” เป็นแกนกลาง และใช้ “กิจกรรมรายสัปดาห์” เป็นกลไกสำหรับสร้าง Dashboard ที่ตอบโจทย์การทำงานจริง

ระบบต้องรองรับการใช้งานทั้ง Desktop, Tablet และ Mobile โดยเฉพาะทีมช่างและทีมติดตั้งที่ต้องใช้งานภาคสนาม เช่น ถ่ายรูปสินค้า บันทึก Serial Number บันทึกผลติดตั้ง แนบ Checklist และอัปเดตสถานะงานทันที

Dashboard ของระบบต้องทำให้ผู้บริหารและทีมปฏิบัติงานเห็นชัดเจนว่า

* สัปดาห์นี้ต้องส่งของที่ไหน
* ต้อง Remote ที่ไหน
* ต้องติดตั้งที่ไหน
* ต้องอบรมที่ไหน
* ใครเป็นผู้รับผิดชอบ
* งานไหนล่าช้า
* งานไหนมีปัญหา
* งานไหนต้องเร่งติดตามเพื่อส่งมอบและเปิดใบแจ้งหนี้

ระบบนี้จะช่วยยกระดับการทำงานจาก Excel Tracking ไปสู่ Operation Management System ที่สามารถติดตามงานได้ครบวงจร ใช้งานได้จริงในสำนักงานและหน้างาน และต่อยอดเป็น KPI ของทีมได้ในอนาคต
